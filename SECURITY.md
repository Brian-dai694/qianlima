# 安全策略 · Security Policy

## 报告漏洞

如果你发现安全问题（尤其是可能导致隐私数据泄露、高危动作绕过治理拦截、或压缩摘要攻击的问题），请**不要**公开开 issue。

请通过 GitHub 私密渠道（Security Advisory）或仓库所有者的联系方式私下报告，包含：
- 问题描述与影响面
- 复现步骤
- 受影响的文件 / 脚本 / 工作流

## 本项目的安全边界

千里马是 **Git-safe 公开模板仓**，安全设计围绕以下原则：

- **隐私边界**：真实 token、账号、客户信息、ASIN 运营数据、成本台账严禁提交。`verify-qianlima.ps1` 在提交前校验，CI 复跑。
- **高危动作拦截**：改价、改预算、删除、外发、写回外部系统默认被运行时策略拦截，需显式二次确认。CI 含负测试断言未确认的高危动作被阻止。
- **压缩攻击防御**：压缩摘要视为工作记忆而非事实来源；高危决策必须回读原始来源。见 `.qianlima/rules/compression-attack-defense.md`。

## 支持的版本

仅对最新公开版本（当前 v2.7.1）提供安全修复。
