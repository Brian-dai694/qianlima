# 千里马企业版说明书：老板

## 您每天看什么

- 公司经营摘要：销售、利润、现金、库存、广告和异常。
- 成果摘要：已完成、未完成、待核验和被拒绝的结果。
- 重大风险：超预算、跨部门、外部发送、失败回滚和治理变更。
- 责任状态：哪个负责人正在处理，什么时候完成。

日常任务以日报或摘要呈现，不展示端口、MCP、密钥、Runner 或技术日志。

## 哪些事情需要您审批

- 超过公司设定金额或影响阈值的采购、付款和预算。
- 跨部门、跨企业的重要数据外发。
- 修改企业治理规则、扩大关键 Agent 权限。
- 重大业务例外或回滚失败。

以下事项通常不需要您逐条审批：任务成果写入、内部项目上传、临时文件清理、部门范围内的可逆操作、有效批量授权内的重复动作。

## 如何审批

审批页面只应显示：要做什么、为什么做、最大影响、负责人、数据范围、预算、有效期、是否可撤销。看不懂或证据不足时选择“退回补充”，不要直接放行。

## 员工和 MCP

您不需要为每次 MCP 调用点击确认。业务负责人可审批员工在限定时间内，使用指定 Agent、指定 MCP、指定工具和指定数据。超范围、到期或员工调岗后自动失效。

## 您不应做什么

- 不把自己设置为日常平台超级管理员。
- 不把企业 Token 发给员工。
- 不审批自己发起的高风险动作。
- 不因为任务紧急而取消证据、预算和回滚要求。

## 异常时

看到“越权、数据外发、回滚失败、税务/海关提交异常”时，先冻结动作，由业务负责人或安全管理员处理。老板负责风险决策，不负责亲自修系统。
