# 千里马企业版说明书：员工

## 您怎么使用

直接用自然语言告诉千里马要做什么，例如：

```text
分析这个产品本周利润下降的原因。
生成本月广告、库存和销售周报。
用已批准的领星只读 MCP 查询这个店铺的订单和库存。
整理这份 Listing 的标题、主图、五点和长描述。
```

系统会自动判断任务等级、可用 Agent、可读取数据和是否需要审批。您不需要输入端口、密钥、Grant 或 JSON。

## 使用 MCP

第一次使用某个 MCP 或扩大工具/数据范围时，系统会发送申请给业务负责人。批准后，员工 Agent 在有效期内经本机 Connector 直接使用指定 MCP。

审批只对以下范围有效：您本人、当前设备、当前 Agent 版本、指定 MCP、指定工具、指定数据、预算和有效期。不能转给其他员工使用。

## 您可以做什么

- 查询本部门和分配项目的数据。
- 生成任务成果、报告和分析。
- 查看自己任务的业务目标、成果和处理待办。
- 对证据不足或结果错误进行反馈。
- 申请新的项目、Agent、MCP 或数据范围。

## 您不能做什么

- 使用他人的账号、Token、Grant 或 MCP 会话。
- 把企业文件上传到未批准的外部服务。
- 绕过 Connector 直接连接企业 MCP。
- 自己批准自己的高风险动作。
- 把“Agent 显示完成”当作业务系统已经写入成功。

## 任务失败怎么办

先看处理端：失败原因、已保留成果、是否可重试、负责人和下一步。不要通过增加权限或反复重试解决越权、数据冲突和写入失败。

## 调岗或离职

调岗后旧部门权限会先失效，再由新主管分配新范围。离职时 Agent、MCP 和凭据会被撤销，未完成任务和成果移交给指定人员；历史贡献仍保留您的署名和审计记录。
