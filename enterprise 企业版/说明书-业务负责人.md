# 千里马企业版说明书：业务负责人

## 您负责什么

- 明确项目目标、部门范围、数据范围和成本中心。
- 决定员工可以使用哪些业务 Agent 和 MCP。
- 审批部门内常规可逆 L4，设置限时、限额批量授权。
- 查看任务五端，分配失败和核心问题的处理人。
- 员工调岗或离职时完成任务与产物移交。

## 新员工

新员工默认只有本部门和分配项目的最小权限，没有 MCP 和企业写入权限。先确认岗位、项目和数据范围，再按需要批准 Agent/MCP，不要复制前任权限。

## 审批员工 MCP

批准时确认七项：员工、设备、Agent 版本、MCP Server、具体工具、数据范围、有效期与预算。审批后员工 Agent 可经本机 Connector 直接连接 MCP，中央 Broker 不进入每次数据流，但每次调用仍受本机门禁检查。

不要批准“全部工具、全部店铺、永久有效、无限预算”。

## 日常审批怎么减负

- 任务成果写入和临时清理：按 L2 自动处理。
- 内部上传、新版本和内部发送：按项目策略处理。
- 常规可逆业务动作：由您或对应责任人审批。
- 重复动作：使用时间窗、次数、金额和总量受限的批量授权。
- 超阈值财务或治理关键事项：升级给老板或安全管理员。

## 报表与计划

- 日报：异常、销售、预估利润、广告、库存和物流。
- 周报：趋势、关键词、转化、补货和供应商进度。
- 月报：正式利润结算、库存账龄、采购物流和计划完成。
- 季报/年报：经营结构、利润、税务海关、产品组合和供应链。

日利润和周利润是经营估算；月、季、年度利润必须使用受控结算口径。

## 员工调岗或离职

调岗先撤销旧部门 Grant 和 MCP，再移交任务，最后分配新权限。离职必须配合 HR 停用身份、撤销 Agent/MCP/凭据、回收设备并指定接手人。
