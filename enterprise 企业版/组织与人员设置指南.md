# 千里马企业版：组织与人员设置指南

老板不需要理解 RBAC、Grant 或 Agent Card。第一次设置只需要准备四项：

1. 公司名称。
2. 老板或业务负责人姓名与员工编号。
3. 部门名称，例如运营、产品、财务。
4. 一名与老板不同的 IT/安全管理员。

运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File ".\enterprise 企业版\new-enterprise-organization.ps1"
```

## 四种角色

| 角色 | 能做什么 | 不能做什么 |
|---|---|---|
| 老板/业务负责人 | 看公司汇总、预算，审批企业业务动作 | 不能审批自己发起的 L4，默认不是平台管理员 |
| 部门主管 | 管理本部门员工、任务和预算 | 不能查看其他部门原始资料 |
| 员工 | 使用批准的 Agent 完成本部门项目任务 | 不能跨部门、不能批准、不能直接企业写入 |
| IT/安全管理员 | 管理设备、Agent、隔离和安全恢复 | 不能替老板批准业务动作 |

## 员工看到什么

员工只需要知道：自己的部门、可用 Agent、当前任务等级和是否需要审批。系统不向普通员工展示端口、密钥、Runner、RBAC 或策略 JSON。

## 默认安全规则

- 新员工默认是“员工”，只访问本部门和分配的项目。
- 老板不会自动成为平台超级管理员。
- IT/安全管理员不能审批业务 L4。
- 员工换部门后，旧部门 Grant 必须撤销并重新计算权限。
- 员工离职后，身份、Grant 和凭据引用立即失效。
- 未指定独立 IT/安全管理员时，只能保存配置草稿，不能企业上线。
