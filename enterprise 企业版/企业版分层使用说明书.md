# 千里马企业版分层使用说明书

本说明书面向第一次使用企业 Agent 的人员。先选择自己的角色，不需要从头阅读全部技术资料。

| 我是谁 | 应阅读 | 日常关注 |
|---|---|---|
| 老板/企业负责人 | `说明书-老板.md` | 公司结果、利润、重大风险、超阈值事项 |
| 业务负责人/部门主管 | `说明书-业务负责人.md` | 项目、员工业务范围、MCP 审批、异常处理 |
| 员工 | `说明书-员工.md` | 发起任务、使用 Agent、查看成果、完成待办 |

## 所有人只需理解四句话

1. Agent 只能处理当前任务批准的数据和工具。
2. 员工 Agent 使用 MCP 前，需要业务负责人批准。
3. 日常可逆操作由责任人处理，不需要老板逐条审批。
4. 任何人离职、调岗、设备异常或权限撤销后，旧会话立即失效。

## 简化任务等级

| 等级 | 普通理解 | 常见例子 |
|---|---|---|
| L0 | 只聊天，不读企业数据 | 解释概念、讨论想法 |
| L1 | 简单只读 | 查看公开资料 |
| L2 | 本部门受控分析 | 生成报告、写任务成果 |
| L3 | 复杂分析和内部协作 | 跨数据分析、内部上传、新版本 |
| L4 | 正式业务状态变化 | ERP 写入、外发、采购、部署、正式申报 |

L4 表示必须受控，不表示所有事项都要老板审批。

## 五个任务视图

- 业务端：为什么做、目标是什么。
- 成果端：产出了什么、是否核验通过。
- 失败端：失败在哪、影响多大。
- 核心问题端：真正原因和未知项是什么。
- 处理端：谁负责、何时完成、下一步是什么。

老板默认看业务端、成果端和重大异常；业务负责人看五端；员工重点看自己的业务摘要、成果和处理待办。
