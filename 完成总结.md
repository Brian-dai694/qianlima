## 下一步开发计划

### 阶段 1：Workflow 执行引擎（核心）
**文件**：`.qianlima/executor/workflow_runner.py`

**功能**：
- 解析 `.qianlima/workflows/*.wf.yaml`
- 执行 workflow steps（data_agent → analysis_agent → execution_agent）
- 调用 MCP 工具获取数据
- 应用诊断规则（如 `daily_ad_report.wf.yaml` 中的 5 条规则）

**优先级**：🔴 高（执行器的核心）

### 阶段 2：MCP 客户端封装
**文件**：`.qianlima/executor/mcp_client.py`

**功能**：
- 封装 pangolinfo MCP 工具：
  - `get_amazon_product(asin, site)` - 获取产品详情
  - `filter_niches(...)` - 筛选利基市场
  - `ai_search(query)` - Google AI 搜索
- 封装 sorftime MCP 工具（关键词排名追踪）
- 数据源认证管理（读取 `secrets.local.yaml`）

**优先级**：🔴 高（数据获取层）

### 阶段 3：模板渲染器
**文件**：`.qianlima/executor/template_renderer.py`

**功能**：
- 读取 `.qianlima/templates/*.md`（Jinja2 模板）
- 渲染数据到模板
- 生成最终报告到 `reports/`
- 支持版本管理（`{date}_{name}_{marketplace}_V{version}.md`）

**优先级**：🟡 中（输出层）

### 阶段 4：成本追踪器
**文件**：`.qianlima/executor/usage_tracker.py`

**功能**：
- 记录 API 调用次数（MCP 工具、Claude API）
- 追踪 Token 使用量
- 估算成本
- 写入 `usage-ledger/{date}_{workflow}_usage.yaml`

**优先级**：🟢 低（可观测性）

### 阶段 5：Claude API 集成（可选）
**文件**：`.qianlima/executor/claude_client.py`

**功能**：
- 用 Claude API 执行分析步骤（analysis_agent）
- 自然语言生成建议
- 应用诊断规则时的推理

**优先级**：🟢 低（增强功能，当前可用 MCP 替代）
