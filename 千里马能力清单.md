# 千里马计划 - 当前能力清单

**更新时间**: 2026-06-28 14:00  
**版本**: v1.0 (MVP 实现完成)

---

## 🎯 核心定位

千里马计划是一套**面向 AI Agent 的高可靠执行与工作治理系统**，让 Agent 能够：
- 连接真实数据源
- 理解工作状态
- 安全可靠地执行任务
- 持续优化改进

### 2. 任务执行器 ⭐ 本次实现

#### 自然语言接口
- ✅ **任务匹配**：从自然语言自动识别任务
  ```
  "我要做竞品对比" → competitor_comparison
  "帮我算利润" → profit_check
  "生成广告日报" → daily_ad_report
  ```

- ✅ **交互模式**：对话式任务执行
- ✅ **CLI 模式**：脚本化批量执行

#### Workflow 执行引擎
- ✅ **步骤编排**：按 workflow 定义自动执行
  - data_agent：数据读取与校验
  - analysis_agent：指标计算与分析
  - execution_agent：报告生成
  - audit_agent：质量验证

- ✅ **数据读取**：
  - 飞书表格（lark_sheet）
  - 领星 ERP（browser）
  - 本地文件（inbox/）
  - MCP 工具（Amazon、Google）

- ✅ **指标计算**（以广告日报为例）：
  - 基础指标：spend, sales, orders, clicks, impressions
  - 衍生指标：ACoS, CPC, CTR, CVR, TACoS
  - 自动处理空值和除零

- ✅ **诊断规则**（5 条内置规则）：
  1. 高花费无单（spend ≥ $10 且 orders = 0）
  2. ACoS 过高（> 目标 ACoS × 1.3）
  3. 表现优秀（ACoS ≤ 目标且 orders ≥ 2）
  4. 高点击低转化（clicks ≥ 15 且 CVR < 0.5%）
  5. 低曝光（impressions < 100 且 spend < $5）

- ✅ **报告生成**：
  - Markdown 格式
  - 核心指标表格
  - 诊断与建议
  - 数据来源追踪
  - 执行统计

#### MCP 工具集成（框架就位）
- ✅ **Pangolinfo 工具**：
  - get_amazon_product(asin) - 产品详情
  - filter_niches(...) - 利基市场筛选
  - ai_search(query) - Google AI 搜索

- ✅ **数据源封装**：
  - read_lark_sheet(...) - 飞书表格
  - 工具调用统计

## 🎨 用户体验

### 大众使用入口
用户可以用**自然语言**直接说明需求，无需理解技术细节：

```
✅ "我要做竞品对比"
✅ "帮我优化这个 Listing"
✅ "算一下这个产品赚不赚钱"
✅ "跑一下这些关键词排名"
✅ "帮我判断这个品类能不能做"
✅ "生成今天的广告日报"
```

### 执行流程（对用户透明）
1. 匹配任务卡
2. 收集必要信息（ASIN、站点、日期等）
3. 自动执行 workflow
4. 生成结构化报告
5. 显示诊断建议

## ⚙️ 技术架构

### 分层设计

```
┌─────────────────────────────────────┐
│      用户自然语言接口                 │
│  "我要做竞品对比" / "生成广告日报"      │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│      任务匹配层 (task_matcher.py)    │
│  plain_language_trigger → task_card  │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│    Workflow 执行层 (workflow_runner) │
│  解析 .wf.yaml → 编排 agent 步骤     │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│     数据获取层 (mcp_client.py)       │
│  MCP 工具 + 飞书 + 领星 + 本地文件    │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│     报告生成层 (内置模板)             │
│  Markdown 表格 + 诊断 + 数据来源      │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│     成本追踪层 (usage_tracker)       │
│  Token 使用 + 工具调用 + 用时统计     │
└─────────────────────────────────────┘
```

## 🔧 当前限制

### 需要配置才能使用真实数据

1. **飞书 API**：
   - 需配置 `secrets.local.yaml`（已有 `secrets.example.yaml` 模板）
   - 需提供 spreadsheet_token 和 sheet_id

2. **MCP 工具**：
   - 当前使用模拟数据
   - 需在 Claude Code 环境中运行以调用真实 MCP 服务器
   - 或实现 MCP 协议客户端

3. **Python 环境**：
   - 需安装 Python 3.11+
   - 需安装依赖：`pip install -r requirements.txt`

## 💡 使用建议

**立即可用**：
```powershell
# 1. 启动工作区（验证骨架）
.\platforms\windows\启动千里马计划.ps1

# 2. 安装 Python + 依赖
pip install -r .qianlima\executor\requirements.txt

# 3. 运行广告日报（使用模拟数据）
python .qianlima\executor\executor.py --task "生成广告日报"
```

**配置真实数据后**：
1. 创建 `.qianlima/secrets.local.yaml`（参考 `secrets.example.yaml`）
2. 配置飞书 API token
3. 运行执行器，获取真实业务数据
